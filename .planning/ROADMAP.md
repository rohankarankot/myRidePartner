# myRidePartner Roadmap

## Overview
**Total Phases:** 5
**Requirements Mapped:** 17

## Phase 1: Authentication & Shared Identity
**Goal**: Establish the multi-app authentication foundation using Google OAuth and isolated AppSources.
**Requirements**: AUTH-01, AUTH-02
**UI hint**: yes
**Success Criteria**:
1. User can successfully sign in via Google OAuth on both mobile and web.
2. The backend correctly identifies and registers the user under the provided `AppSource`.
3. Authentication sessions persist securely across app restarts.

## Phase 2: Core Trip Lifecycle
**Goal**: Enable the end-to-end trip creation, discovery, and joining process.
**Requirements**: TRIP-01, TRIP-02, TRIP-03, TRIP-04
**UI hint**: yes
**Success Criteria**:
1. Users can create a trip specifying start and end locations.
2. Users can browse a list of available trips and submit join requests.
3. Trip creators can approve or deny pending join requests.

## Phase 3: Real-Time Communication
**Goal**: Implement Socket.io based messaging and Cloudinary media uploads for trip and community chats.
**Requirements**: MSG-01, MSG-02, MSG-03
**UI hint**: yes
**Success Criteria**:
1. Users can send and receive real-time text messages within an active trip.
2. Users can send and receive real-time text messages within a community group.
3. Users can successfully upload images to Cloudinary and share them in chats.

## Phase 4: Communities & Notifications
**Goal**: Expand social features with community groups and ensure users are alerted to important events via push notifications.
**Requirements**: COMM-01, COMM-02, NOTF-01, NOTF-02
**UI hint**: yes
**Success Criteria**:
1. Users can create and join location/interest-based community groups.
2. The system sends reliable push notifications for new trip requests and chat messages.
3. Notifications successfully wake the app or display in the system tray.

## Phase 5: Administration & Polish
**Goal**: Complete the user profile capabilities, implement the backoffice dashboard for moderation, and integrate AdMob monetization.
**Requirements**: AUTH-03, ADMN-01, ADMN-02, ADMN-03, ADMN-04, AD-01
**UI hint**: yes
**Success Criteria**:
1. Administrators can view system metrics and manage user accounts from the web dashboard.
2. Administrators can moderate reports and oversee trips.
3. Mobile users can edit their profile details.
4. Mobile application successfully loads and displays AdMob advertisements.
